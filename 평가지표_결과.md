## 1. AI 모델 성능 평가

### 1.1 KLUE 시설 분류 모델

**실험 목적**: 구리시 시설을 7개 카테고리로 자동 분류하는 최적 모델 선정

**카테고리**: school, hospital, restaurant, market, sports, apartment, cafe

**모델 비교 실험 결과**:

| 모델 | Accuracy | Precision | Recall | F1-score |
|------|----------|-----------|--------|----------|
| KoBERT | 0.949 | 0.798 | 0.844 | 0.817 |
| KoELECTRA | 0.934 | 0.921 | 0.920 | 0.359 |
| **KLUE** | **0.938** | **0.863** | **0.865** | **0.863** |

**최종 모델 선정**:
- **KLUE 선정 (F1-score 0.863)**
- Precision과 Recall의 균형이 가장 우수
- 7개 카테고리 분류에서 안정적인 성능

---

## 2. 시스템 구성

### 2.1 AI 기술 스택

본 프로젝트는 다음 AI 기술들을 결합:

1. **KLUE (klue/roberta-base)** - 시설 카테고리 분류
2. **GPT-4o-mini (OpenAI)** - 자연어 이해 및 응답 생성
3. **Ko-sRoBERTa** - 시설명 임베딩 (Sentence Transformers)
4. **RAG** - SQLite + ChromaDB 하이브리드 검색

### 2.2 시스템 흐름

```
사용자 입력
    ↓
FastAPI (LLM Parser)
    ↓
KLUE 모델 (시설 분류)
    ↓
RAG 검색 (SQLite + ChromaDB)
    ↓
GPT-4o-mini 응답 생성
    ↓
사용자에게 결과 반환
```

## 3. 주요 기능

### 3.1 시설 카테고리 자동 분류
- KLUE 모델이 시설명을 7개 카테고리로 분류
- F1-score 0.863의 높은 정확도

### 3.2 시설 기반 아파트 추천
- 입력 예시: "구리고 반경 500m 아파트 추천해줘"
- 구리고 좌표 기준 500m 이내 아파트 검색
- 거리순 정렬하여 결과 제공

### 3.3 후속 질문 이해
- "가장 가까운 곳은?" → 최단거리 아파트 제공
- "몇 개나 있어?" → 검색 결과 개수 반환
- chat_memory를 통한 대화 컨텍스트 유지
